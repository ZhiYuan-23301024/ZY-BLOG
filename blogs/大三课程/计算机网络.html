    # 同步/异步传输

    同步异步指的是时间间隔是否固定

    • 同步传输方式：无字节间隔，无起始和中止位，要求双方对齐时间长度。
      • 外同步：利用外部时间线发送时钟信号，实现同步
      • 内同步：将时钟信息编码到发送数据中，如曼彻斯特编码和差分曼彻斯特编码
    • 异步传输方式： 以字节为处理单位，但字节之间的时间间隔不固定，但是字节里的比特是同步的

    # 什么是低频

    “可不可以解释一下什么叫做频率较低的信号，我的理解中信号只有呈周期变化的时候才有所谓的频率一说吧，但是基带信号并不是呈周期性变化的啊” 这里的核心思想是：​任何一个信号，无论它多复杂，是否周期变化，都可以被看作是由许多个不同频率、不同幅度的基本正弦波“叠加”而成的​。而分析一个信号中包含哪些频率的正弦波成分（即“频率成分”），以及每个成分的“强度”是多少，就是傅里叶分析所做的事情 ​概念的扩展​：通过傅里叶分析，​​“频率”的概念被扩展为描述任何信号能量在频域上的分布情况。我们说的“低频信号”，通常指的是其主要频率成分分布在低频段

    • 码元：信号的编码单位，在使用时间域的波形表示信号的时候，代表不同离散数值的基本波形被称为码元

    # 常用编码方式

    • 不归零制 信号在每个码元期间不会回到零电平，效率最高，存在同步问题
    • 归零制 信号在每个码元期间都会回到零电平，将时钟信号参与编码，自同步信号，效率低
    • 曼彻斯特编码 在每个码元的中间时刻电平发生跳变，该跳变既表示时钟信号也表示数据，自同步信号，效率更高
    • 差分曼彻斯特编码
      基于曼彻斯特编码，但是中间时刻电平变化仅表示时钟信号，抗干扰

    [!NOTE] 多播如何工作

    1. 1.主机加入组播组​：当一个应用程序（如视频会议客户端）想要接收某个多播组的数据时，它会通过操作系统加入一个特定的多播组（也就是监听一个特定的多播地址，例如 224.0.1.1）。这相当于告诉本地网络：“我对发往这个组的数据感兴趣”。

    2. 2.数据发送​：发送方只需将数据包的目标地址设置为这个多播组地址，然后发送一次即可。发送方自己并不需要加入这个多播组。

    3. 3.​网络智能转发​：支持多播的路由器会通过特定的多播路由协议（如PIM）来交换信息，了解网络中有哪些主机加入了哪些多播组。当收到多播数据包后，路由器会只向有组成员存在的网络分支进行转发和必要的复制，而不会将数据包泛滥到所有网络。这是多播相比广播更加高效和智能的地方。

    4. 4.本地接收与过滤​：在局域网内，主机的网络接口卡（NIC）和IP层会进行过滤。它们会检查数据包的目的地址，只接收那些目标地址是自己已加入的多播组的数据包，而忽略其他多播流量

    # 划分子网编址方法

    改进分类编址造成的大量地址浪费问题 从分类子网的主机号部分划出子网编号区域，实现划分子网

    • 子网掩码：
      • 32bit
      • 1位对应网络位，0位对应主机号位
      • 默认子网掩码： 未划分子网的情况下使用的子网掩码 主机号部分全0就好了

    • 客户端端口号（短暂端口号） 不同计算机之间的端口号/不同协议之间的端口号没有任何关联 发送方复用&接收方分用 不同协议使用同一个IP发出来，将同一个IP数据报中的数据按照不同的协议区分开

    # UDP & TCP

    对发送时间的限制 UDP支持单薄，多播和广播 TCP只支持单播 对应用层报文处理 数据传输可靠性 首部结构对比（UDP固定8B）

    # TCP报文首部格式

    • 源端口
    • 目的端口
    • 序号：32bit，用于指出本TCP报文段数据载荷的第一个字节的序号（TCP把要发送的整个数据看作一个连续的字节流。TCP为这个字节流里的每个字节都按顺序编上号。）
    • 确认号：对之前受到数据的确认，可指出下一个TCP报文段序号的期望值
    • ACK：取值为1时，确认号字段才有效
    • 数据偏移：4bit，取值以4Byte为单位，指出TCP报文段数据载荷部分起始处到TCP报文段起始处有多远，实际上指的是TCP报文段的首部长度
    • 保留：6bit
    • 窗口：16bit，取值以1Byte为单位，指出报文发送方的接收窗口大小（流量控制）
    • ==检验和==：16bit，检查整个TCP报文段在传输过程中是否出现误码
