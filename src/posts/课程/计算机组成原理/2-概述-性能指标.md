## CPU指标

### 时钟周期

原始模拟信号->数字脉冲信号
用于计算机内部组件同步

```ad-note
title:什么是同步什么是异步

**同步**：通过时钟信号协调同步（这个同步指协作）
**异步**：不通过时钟信号，而是通过通信协定，收发信号进行协作
```

### 主频

>时钟频率，CPU一秒钟可以执行多少个时钟周期

$$
Frequency=\frac{1}{Clock~Cycle}
$$
### 指令执行指标

- **CPI**：指令周期，执行一条指令需要的平均时钟周期。一个指令的执行需要若干个基本操作完成。基本操作的周期是机器周期。

- **IPC**：每周期平均指令数
$$
IPC=\frac{1}{CPI}
$$

- **IPS**：每秒指令数
$$
IPS=\frac{主频}{CPI}
$$
- **MIPS**：每秒百万指令数

### 浮点运算指标

**FLOPS**：每秒浮点运算次数

## 计算机指标

### 字长（机器字长）

CPU一次能处理的数据长度
$$
内部寄存器宽度=ALU位数=内部总线宽度=机器字长
$$
越大，一次理论上的计算能力就更厉害

出现这个现象的原因是：**`code-server` 是一个重度依赖 WebSocket 的应用，而在你输入密码登录时，正好触发了 WebSocket 连接升级。Cloudflare 的免费临时隧道（Quick Tunnel）对 WebSocket 的支持不太稳定，或者它没有正确地将必要的请求头转发给 `code-server`，导致连接被中断，从而抛出 502 错误。**

要解决这个问题，我们有两种方法。建议你先尝试**方案一**，如果不行再使用更稳定的**方案二**。

### 方案一：调整 `code-server` 启动参数（最简单）

我们可以通过修改 `code-server` 的启动方式，让它更“亲和”反向代理，并排除一些潜在的干扰。

**1. 先停止当前运行的 `code-server`。**

**2. 使用以下参数重新启动它：**

```bash
code-server --bind-addr 0.0.0.0:8080 --auth none
```

- `--bind-addr 0.0.0.0:8080`：确保它监听在所有网络接口上，避免因 Docker/Termux 网络隔离导致的连接失败。
    
- `--auth none`：暂时禁用 `code-server` 自带的密码验证。既然要通过公网访问，我们可以把安全交给 Cloudflare 来处理（或者等你用本地测试通了再加上密码）。
    

**3. 保持你的 Cloudflared 隧道运行不变，再次尝试访问。**

---

### 方案二：放弃不稳定的临时隧道，改用免费的“命名隧道”（推荐 ⭐⭐⭐）

Cloudflare 给免费账户提供了 **Zero Trust** 功能，里面的 **Named Tunnel（命名隧道）** 对 WebSocket 的支持非常好，而且完全免费。虽然步骤稍微多一两步，但一劳永逸。

**1. 注册/登录 Cloudflare 账号**

访问 [Cloudflare Zero Trust 面板](https://one.dash.cloudflare.com/)，使用邮箱免费注册一个账号。

**2. 创建命名隧道**

- 在左侧菜单找到 **Networks（网络）** -> **Tunnels（隧道）**。
    
- 点击 **Create a tunnel（创建隧道）**。
    
- 给隧道起个名字（比如 `termux-code`），然后点击 **Save（保存）**。
    
- 在下一页的 **Connectors（连接器）** 部分，Cloudflare 会给你提供一段类似这样的命令：
    
    ```bash
    cloudflared tunnel login
    cloudflared tunnel route dns <你的隧道UUID或名字> code.yourdomain.com
    ```
    
    _(注：你不需要自己买域名，Cloudflare 会分配一个免费的 `*.cfargotunnel.com` 的随机子域名，或者你可以用自己的域名)_。
    

**3. 在 Termux 中执行配置**

直接在 Termux 里依次运行刚才页面上显示的命令。这会将你的 Termux 认证到你的 Cloudflare 账号下。

**4. 修改本地隧道配置**

Cloudflare 网页上会提供一个 **Public Hostname（公共主机名）** 的配置项：

- **Subdomain（子域名）**：随便填（比如 `code`）
    
- **Domain（域名）**：选择 Cloudflare 分配给你的免费子域名（或者你自己的域名）
    
- **Path（路径）**：留空
    
- **Type（类型）**：`HTTP`
    
- **URL**：`http://localhost:8080`
    

点击 **Save（保存）**。

**5. 重新运行隧道**

现在，你不再需要使用命令行里的 `--url` 参数了，直接运行：

```bash
cloudflared tunnel run
```

Cloudflare 会自动读取你刚才在网页上的配置，并建立一条**专为 WebSocket 优化过的稳定隧道**。

**6. 访问测试**

使用 Cloudflare 分配给你的新域名（类似 `https://code.xxxxx.cfargotunnel.com`）访问，此时输入密码应该就能顺利进入桌面了。

你可以先试试**方案一**，如果还是不行，那就花 5 分钟走一遍**方案二**，方案二绝对是解决这类问题的最完美方案。
